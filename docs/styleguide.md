# Walking App – Visual Style Guide (MVP)

## 1. Design Principles

The visual style is built around a **modern outdoor aesthetic**, combining:

* natural colour inspiration (earth, foliage, terrain)
* soft, approachable UI elements
* strong readability in bright outdoor conditions

The interface balances:

* **functional clarity** (stats, tracking, actions)
* **visual immersion** (maps, photography, terrain)

---

## 2. Colour System

### Primary Palette

* **Primary (Accent / Action)**

  * Burnt Orange `#E65100`
  * Used for primary actions (Start, Follow, CTA buttons)

* **Secondary (Success / Active State)**

  * Natural Green `#2E7D32`
  * Used for active tracking, success states, confirmations

* **Tertiary (Utility / Secondary UI)**

  * Deep Teal `#006064`
  * Used for icons, secondary controls, subtle highlights

* **Neutral (UI Foundation)**

  * Slate Grey `#455A64`
  * Used for text, backgrounds, containers

### Usage Principles

* Primary colour = **intent and action**
* Green = **status and confirmation**
* Teal = **supporting UI elements**
* Neutral = **structure and readability**

Gradients and tonal steps are used within each palette for depth and hierarchy.

---

## 3. Typography

### Hierarchy

* **Headline**

  * Large, bold, highly legible
  * Used for titles and key screens (e.g. walk name)

* **Body**

  * Medium weight
  * Used for descriptions and supporting text

* **Label**

  * Small, uppercase or spaced
  * Used for metadata (distance, time, etc.)

### Characteristics

* Clean, modern sans-serif
* High contrast for outdoor readability
* Generous spacing for clarity at a glance

---

## 4. Layout & Structure

### Core Layout Style

* **Card-based design**
* Rounded containers
* Soft spacing between sections
* Clear visual grouping of data

### Key Patterns

* Stats presented in **grid cards**
* Content grouped into **distinct panels**
* Strong vertical flow for mobile usability

### Spacing

* Generous padding for touch interaction
* Consistent margins between components
* Clear separation between functional areas

---

## 5. Components

### Buttons

* **Primary**

  * Filled (orange)
  * Rounded
  * Strong visual weight

* **Secondary**

  * Muted or tonal
  * Used for less critical actions

* **Outlined**

  * Minimal emphasis
  * Used for alternative options

### Status Indicators

* “Tracking Active” uses:

  * soft green background
  * subtle pill shape
  * low visual noise but clear meaning

### Cards

* Light background
* Rounded corners
* Subtle elevation (or contrast)
* Used for:

  * stats
  * routes
  * summaries

---

## 6. Map & Visual Layer

### Map Style

* Terrain-focused
* Emphasis on:

  * elevation
  * natural features
  * paths over roads

### Route Display

* Route lines are:

  * bold
  * high contrast (blue/orange)
  * clearly visible over terrain

### Imagery

* High-quality landscape photography
* Used as:

  * hero backgrounds
  * route previews

---

## 7. Interaction Design

### Primary Actions

* Always clearly visible
* Positioned near thumb reach
* Strong contrast from background

### Recording Screen

* Focus on:

  * live stats
  * large readable timer
  * simple controls (pause / stop)

### Feedback

* Minimal but clear:

  * colour changes
  * subtle animations
  * vibration/audio (for route alerts)

---

## 8. Visual Tone

The overall tone is:

* **Calm**
* **Natural**
* **Reliable**
* **Practical**

It avoids:

* overly technical visuals
* cluttered interfaces
* excessive colour noise

Instead, it prioritises:

* clarity in outdoor use
* quick understanding at a glance
* a sense of exploration and place

---

## 9. Design Summary

The design combines:

* **functional tracking UI** (stats, controls)
* **map-based exploration**
* **lifestyle-inspired visuals**

to create a walking app that feels:

* intuitive to use
* visually grounded in nature
* focused on real-world use rather than abstract data
